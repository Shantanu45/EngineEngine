Forward Renderer PBR Plan
=========================

Current State
-------------
- CPU material data already follows the glTF metallic-roughness shape:
  - base color factor/texture
  - metallic factor
  - roughness factor
  - metallic-roughness texture
  - normal texture/scale
  - occlusion texture/strength
  - emissive texture/factor
  - alpha mode/cutoff
- GLTF material import already maps `PBRMaterial` into `Rendering::Material`.
- The forward shader samples PBR inputs, but lighting is currently a
  metallic-roughness Blinn approximation rather than a Cook-Torrance BRDF.
- The forward shader has a simple ambient term and no image-based lighting.


Milestone 1: Direct-Lighting PBR
--------------------------------
Replace the current Blinn-style BRDF in `assets/shaders/lib/lighting.glsl`
with a Cook-Torrance metallic-roughness BRDF.

Tasks:
- Add GGX / Trowbridge-Reitz normal distribution.
- Add Smith geometry term.
- Add Schlick Fresnel.
- Use Lambert diffuse scaled by `(1 - metallic)`.
- Keep existing light structs, material UBO layout, texture bindings, and
  forward renderer pipeline unchanged.
- Preserve current directional, point, and spot light dispatch.
- Clamp roughness to a safe minimum to avoid unstable highlights.

Validation:
- Compare roughness changes on a material sphere or known glTF PBR sample.
- Check Sponza still renders without shader or descriptor regressions.


Milestone 2: Alpha Mask Support
-------------------------------
The material system carries alpha mode and cutoff, but the shader does not yet
apply alpha behavior.

Tasks:
- Sample base color alpha.
- Multiply by `base_color_factor.a`.
- Discard fragments for masked materials below `alpha_cutoff`.
- Leave blended transparency for a later transparent pass.

Notes:
- This likely needs a material flag or shader-side value that exposes alpha mode
  to GLSL. Do not build a full transparent renderer in this milestone.


Milestone 3: Ambient And AO Cleanup
-----------------------------------
Replace the current hardcoded ambient approximation with a PBR-safe placeholder
until image-based lighting exists.

Tasks:
- Apply ambient as `baseColor * ambient_strength`.
- Apply occlusion through `mix(1.0, ao, occlusion_strength)`.
- Avoid double-applying base color between ambient and direct lighting.
- Keep ambient simple and deterministic.


Milestone 4: Image-Based Lighting
---------------------------------
Add environment lighting after direct PBR is stable.

Tasks:
- Generate or load diffuse irradiance cubemap.
- Generate or load prefiltered specular environment cubemap.
- Add BRDF LUT texture.
- Extend `RenderResourceStore` or a small environment resource helper to own
  these resources.
- Add bindings to the forward material/global descriptor layout.
- Integrate diffuse IBL and specular IBL in the PBR shader.

Notes:
- Do not start here. IBL changes resource ownership, shader descriptors, and
  preprocessing/loading policy.


Milestone 5: Normal/Tangent Robustness
--------------------------------------
The forward vertex shader assumes valid tangents.

Tasks:
- Detect missing tangents during GLTF import.
- Either generate tangents or mark meshes/materials to skip normal mapping.
- Add shader fallback for invalid tangent basis if needed.


Milestone 6: Transparent Materials
----------------------------------
Handle glTF `BLEND` materials after opaque and masked PBR are correct.

Tasks:
- Split opaque/masked and transparent render instances.
- Add transparent forward pass ordering.
- Add blend pipeline state variants.
- Decide sorting policy.


Suggested Implementation Order
------------------------------
1. Implement Cook-Torrance direct lighting in `lighting.glsl`.
2. Fix forward shader ambient/AO handling enough for direct PBR.
3. Add alpha mask support.
4. Validate with a known glTF PBR asset.
5. Plan IBL resource ownership and descriptor changes.
